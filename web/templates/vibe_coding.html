💡 What is vibe coding?

Loose term for low-ceremony, idea-first coding — describe what you want in plain English, let the AI write the heavy stuff, iterate fast. The point of this workshop is to feel the loop: prompt → working code → run → improve → repeat, until you've shipped something real in under 15 minutes.

Everything here is driven from the terminal with claude — the Claude Code CLI — pointed at the internal local LLM. The prompts work just as well with Copilot or any other agent; the IDE doesn't matter, the loop does.

🧭
How to use this page Open your terminal, run claude in a fresh directory, then click Copy on each prompt below and paste it into the Claude Code chat. That's the entire workflow.

Part 2 — Weather API client
⏱️ 3–4 min

Time to pull real data from the internet. You'll need a free OpenWeatherMap API key — the step-by-step is below if you don't already have one.

setup How to get a free OpenWeatherMap API key
  1. Go to openweathermap.org/api and click Sign In (top-right). New user? Click Create an Account instead.
  2. Sign up with any email — it's a public service, you don't need a corp email.
  3. Confirm the email link they send you.
  4. Sign in → click your username (top-right) → My API keys.
  5. A default key is created for you automatically. Click copy next to it.
  6. Heads up: new keys take a few minutes (sometimes up to ~2 hours) to activate. Quick test once you have it:
    curl "https://api.openweathermap.org/data/2.5/weather?zip=10001,US&appid=YOUR_KEY"
    HTTP 200 → ready. HTTP 401 with "Invalid API key" → wait a bit longer and retry.

Free tier gives you 60 calls/minute and 1M/month — way more than this workshop will use.

Handling it: don't paste the key into code that gets committed. When Claude Code asks for it, drop it in a local .env file or pass it as an environment variable. The agent already knows the pattern — just say "use an env var for the API key".

Stretch 2 — Webex bot integration

You'll need two values from Webex — an access token for a bot you own, and the room/space ID you want it to post into. Step-by-steps below; expand the one you need.

setup How to create a Webex bot & copy its access token
  1. Sign in to developer.webex.com/my-apps with your Webex account.
  2. Click Create a New App → Create a Bot.
  3. Fill in:
    • Bot name — display name shown in chat.
    • Bot username — must end in @webex.bot and be globally unique. If your first choice is taken, add a suffix.
    • Icon — pick a default or upload one (any 512×512 PNG works).
    • Description — short blurb.
  4. Click Add Bot.
  5. The confirmation page shows a Bot Access Token. Copy it now — it's only displayed once. If you lose it, you can regenerate (which invalidates the old token).

Handling it: treat the token like a password. Put it in .env or an env var, never paste it into committed code. Anyone who has it can post as your bot.

setup How to find the room (space) ID to post into

Webex room IDs are long base64-encoded strings — you'll get them via the API.

  1. In the Webex app (desktop or web), add your new bot to the space you want it to post into. Easiest path for testing: create a 1:1 space with the bot by DMing it from your own account — that auto-creates a space.
  2. Open the Webex API explorer at developer.webex.com/docs/api/v1/rooms/list-rooms.
  3. Make sure you're signed in (top-right). Use either your own Webex account or swap the token to your bot's token (top of the page → click the token chip). Bot tokens only see rooms the bot is a member of, which is usually what you want.
  4. Click Run. The response is JSON listing every space.
  5. Find your space by title and copy its id field. That's webex_room_id.

Quick sanity check from the terminal (replace both values):

curl -H "Authorization: Bearer YOUR_BOT_TOKEN" -H "Content-Type: application/json" -d '{"roomId":"YOUR_ROOM_ID","text":"hi"}' https://webexapis.com/v1/messages

HTTP 200 with a JSON body → wired up. HTTP 404 on room → the bot isn't a member of that space; add it and retry.

Prompt
I have a Webex bot access token. Can you modify the weather scheduler to send the weather updates directly to my Webex space instead of just printing them? Send it as plain text messages.

💡 Demo tips

Before you start

  • Have your OpenWeatherMap API key ready in your clipboard.
  • Have your Webex bot token + room ID ready if you plan to do stretch goals.
  • Pre-open a browser tab on timeanddate.com/timer set to 15:00.
  • Confirm Claude Code is actually pointed at the local LLM — run claude, ask "what model are you?", and verify the answer matches the expected internal model.

During the demo

  • Don't fight the agent. If it goes sideways, paste the next prompt and keep moving — vibe coding is about momentum.
  • Show the diffs Claude Code proposes before approving them. The audience sees AI and human-in-the-loop, which is the whole point.
  • Read the prompt out loud as you paste it. The audience needs to understand what you asked for to appreciate what came back.

If the model stalls or returns junk

  • Type /clear to drop context, then re-paste the last prompt with one more sentence of guidance.
  • If the local LLM is slow, drop the temperature down in your Claude Code config or fall back to a smaller model — see /claude-code-with-local-models for the routing options.
