How Does Pokedex Work?

A 15-minute tour of every Pokedex capability — ticket triage, threat intel, multi-tool investigations, write actions, and guardrails.
🤖

1. What is Pokedex?

A SOC AI assistant that lives in Webex and answers in plain English. Behind the chat box: a tool-calling agent loop wired to 26+ security platforms — EDR, SIEM, SOAR, threat intel, identity, the wiki, 32K+ historical tickets — and the discipline to know which one to reach for. One chat box; everything an analyst would otherwise tab through.

🔁

2. The agent loop

Every message runs the same loop: the model reads what you said, decides which tool (or tools) to call, calls them, reads the results, decides if it has enough to answer — and either answers or loops again. Hard caps stop runaway loops: MAX_PER_TOOL_CALLS=2, MAX_SEARCH_CALLS=3, TOOL_RESULT_MAX_CHARS=4000. The model is always grounded in actual tool output, never freewheeling on training data.

🧰

3. The toolbox

~60 tools spanning every layer of the SOC:

  • Tickets & cases: XSOAR, XSIAM, ServiceNow ITSM
  • Endpoint: CrowdStrike Falcon, Tanium, Varonis
  • Network & SIEM: QRadar (AQL), Vectra NDR, the corporate proxy
  • Threat intel: VirusTotal, urlscan, AbuseIPDB, Shodan, Recorded Future, abuse.ch, IntelX, HIBP
  • Identity: Active Directory, on-call staffing, SOC contacts
  • Knowledge: wiki RAG, 32K+ XSOAR ticket index, contacts RAG
  • Write actions: add note to XSOAR ticket, URL block (via XSOAR playbook), AttackIQ BAS triggers
🎯

4. Two-stage routing

Loading all 60 tools into one prompt would blow the context window and confuse the model. So a fast router LLM (Qwen3-8B) peeks at your question first and picks the right tool subset — usually 4–8 tools. The analysis LLM (GLM-4.7-Flash) then runs the actual agent loop with just those. Faster, sharper, fewer wrong-tool calls.

📚

5. Memory & institutional knowledge

Two layers. Per-session memory lets you say "remember this" or "what did I just tell you?" and have it stick across follow-ups. RAG over 32K+ closed XSOAR tickets means Pokedex can answer "how did we handle the last 3 Qakbot tippers?" by retrieving the actual closure notes, not guessing. The wiki and the SOC contact directory are RAG-indexed the same way.

🛡️

6. Guardrails — what Pokedex won't do

  • Off-topic refuser: "Help me with my taxes" → "I'm a SOC security assistant…" — deterministic canned reply, no LLM round-trip, no jailbreak vector.
  • No destructive ops: disable EDR, delete users, mass quarantine — refused. Isolation is gated behind explicit confirmation cards.
  • Blast-radius check: block requests for owned domains or wildcards are auto-refused before they ever reach XSOAR.
  • Credential redaction: even if a password leaked into a closed ticket, RAG retrieval scrubs it before it hits the prompt.
  • Confirmation cards on write actions: URL blocks and notes don't auto-fire — an analyst taps Confirm in Webex.
⚡

7. Infra

Models: GLM-4.7-Flash 8-bit (analysis · tool-calling) and Qwen3-8B (router) on Apple Silicon via vllm-mlx. Embeddings: Qwen3-Embedding-8B on a second Mac Studio. Orchestration: a Python agent loop on lab-vm1 that talks Webex on one side, the model + tools on the other. All on-prem, all the time — no customer data ever leaves the network.

🎤

The bottom line

Pokedex turns a 20-minute, 4-tab investigation into a 90-second chat. Same data, same rigor — minus the tab fatigue.
