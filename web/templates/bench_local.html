🧰 Add a new run

One JSON file per run, plus one entry in the index. Both live in docs/benchmarks/ so they ship in git alongside the markdown summary.

# 1. Run the bench (example: ToolCall-15 against a vllm-mlx endpoint)
cd /tmp/ToolCall-15
export MLX_HOST=http://<mac>.the-company.com:8003
export MLX_API_KEY=<bearer>
export LLM_MODELS="mlx:<model-id>"
node dist-cli/cli/run.js --temperature 0 --timeout 180 --json > /tmp/raw.json

# 2. Strip the progress prefix and save under docs/benchmarks/results/
tail -n +$(grep -n '^{' /tmp/raw.json | head -1 | cut -d: -f1) /tmp/raw.json \
    > docs/benchmarks/results/$(date +%F)-<bench>-<model>-<host>.json

# 3. Append a record to docs/benchmarks/index.json with the matching
#    result_file path, headline scores, host/model labels, and notes.
