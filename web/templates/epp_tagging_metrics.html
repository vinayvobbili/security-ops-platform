💡 How Ring Tagging Works

Three bots automatically assign deployment ring tags across our CrowdStrike and Tanium fleet, so software rolls out in safe, phased waves. 🚀

🤖 The three bots — one per platform

🦅
Jarvis
CrowdStrike hosts
🖧
Case
Tanium On-Prem hosts
☁️
Tars
Tanium Cloud hosts

All three follow the same 5-step pipeline — only the platform endpoints differ.

🔄 The flow, end to end

  1. 📥 Fetch Pull hosts that are missing a ring tag — Jarvis from the CrowdStrike API (OAuth2), Case/Tars from the Tanium GraphQL API (token auth).
  2. 🗂️ Enrich Look each host up in ServiceNow CMDB (cmdb_ci_computer) for its environment, country, and lifecycle — this is what makes the tag correct.
  3. 🧠 Decide Workstations spread across Rings 1–4 by region (phased %); servers get their ring by environment (Dev/Lab→1, Test/QA→2, Stage/DR→3, Prod→4). A hostname-prefix fallback guesses the country if CMDB is empty.
  4. 🏷️ Apply Write the tag back — Jarvis as FalconGroupingTags/… in CrowdStrike, Case/Tars via a bulk Tanium action.
  5. 📊 Report Post success/fail counts + an Excel detail file to Webex, and log the run to the history database behind this dashboard.

⏰ Where & when it runs

  • 🖥️ Always-on services on a VM in the Cyber lab.
  • 🗓️ Scheduled Mon & Thu at 04:00, 12:00, 20:00 ET.
  • 🛑 A safety window + STOP button in Webex lets anyone abort the CrowdStrike run before tags apply.
  • 🙋 Bots also take manual Webex commands for on-demand reports or tagging.

🛠️ Why a host ends up untagged

  • 🗂️ Missing CMDB data — no environment/country to derive a ring from (the most common reason).
  • 🔌 No CMDB entry for the host at all.
  • ⚠️ API error applying the tag.

The "🔍 Untagged Hosts Breakdown by Reason" chart above splits these out for any time range.
