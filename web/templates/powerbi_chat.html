The problem it solves

Power BI datasets carry answers leadership asks for every week, but the queries require DAX — a language only a few PBI Engineers write fluently. Every ad-hoc question became a ticket, a handoff, and a ~30-minute wait. Power BI Explorer puts a natural-language front door in front of the same data, governed by the same authentication and dataset permissions. It removes a skill gate, not a security gate.

🗂️

1. Pick a dataset

The sidebar lists all datasets in the CASM workspace (currently 89). Pick one and the explorer pulls the full schema — every table and every column — plus an auto-built dashboard, with no manual setup.

📊

2. The auto-dashboard

Behind the scenes, the explorer inspects the dataset schema, picks the high-cardinality dimensions and numeric measures, and generates KPI cards plus 4–8 Chart.js visuals — distributions, trends, top-N, coverage gaps. Most routine questions are already answered before anyone types anything.

💬

3. Ask in English

For the deeper questions, type in plain language. A local LLM reads the compact schema, identifies the relevant tables and columns, writes the DAX query, executes it against the live Power BI dataset via the XMLA endpoint, and then explains the result in a sentence or two with the underlying data table.

📤

4. Export in one click

Any result table exports to a professionally formatted Excel with auto-sized columns. The full dashboard exports to PowerPoint using the company template — KPI cards, chart summaries, gold accent bar, navy title strip — ready to drop into a leadership deck.

⚙️

How the numbers are derived

~30 sec vs. ~30 min: The old path — locate the right engineer, context-switch, write DAX, run, screenshot, reply — averages ~30 min of wall-clock time. The explorer returns a live, cited answer in seconds.
~7 hrs / wk reclaimed: Conservative estimate of ~15 ad-hoc PBI requests per week × ~29 min saved each = ~7.25 hours / week of engineering time. At typical loaded rates, that's ~$35k / year in reclaimed capacity, plus the requesters get their answers on their own schedule.
Zero incremental license cost: Runs on our existing local LLM infrastructure — no per-seat NL-to-SQL vendor.

🔒

Access & governance

Auth is the same SSO as the rest of the web app. The Power BI XMLA endpoint uses certificate-based authentication as the existing CASM service principal, so dataset-level Row-Level Security and workspace permissions remain in force. Every query is logged with the requester's identity via the standard web activity log.
