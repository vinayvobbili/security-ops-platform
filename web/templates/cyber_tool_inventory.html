The problem it solves

Answers about "what tools do we own?" live in a spreadsheet on the Security Policy team's laptop. Every "do we have an EDR?", "who sponsors Splunk?", "which tools cover identity?" turns into a ping, a context-switch, and a wait. This page is a self-serve front door over the same catalog — anyone with web-app access can answer their own question in seconds.

🔎

1. Search & sort

The search box matches across every visible field — app name, long name, leader, working client, sponsor — live as you type. Click any column header to sort, click again to flip direction; the table remembers your sort and search query so the view comes back the way you left it.

💬

2. Ask in plain English

The chat widget in the bottom-right has the entire inventory loaded as context. Ask things like "how many tools do we have?", "which tools look like EDR?", "group by sponsor and show counts", "any identity tools?" — the LLM filters, groups, and summarizes against the live list, no ticket needed.

🧪

3. Phase 1 today — Phase 2 later

Today (Phase 1): EAI ID and Application Name are the real values from the Security Policy snapshot. The four descriptor columns — Application Long Name, Sr. Business Leader, Business Working Client, Business Sponsor — are obviously-fake Marvel/DC names as a placeholder, so no one mistakes them for real ownership data. Phase 2 (when EAI provides an API key): the loader swaps to the live API, the synthetic fields are replaced by real ownership, and nothing else on this page changes.

🔒

Access & governance

Auth is the same SSO as the rest of the web app — no extra access path, no separate permissions to grant. The catalog is a read-only view; updates still flow through the Security Policy team. This page removes a skill gate (knowing where the spreadsheet lives), not a security gate.
