🎣 How phishing sentiment analysis works

📥
1. Submit the email — or just a file
Paste the raw source (headers + body), upload the reported .eml, or attach a suspicious file on its own with 📎 Attach file(s). Nothing is stored — refresh the page and it's gone.
🧠
2. Local-LLM sentiment & social-engineering read
An on-prem model (no cloud, no per-token cost) scores tone, urgency and the manipulation tactics in play, names the pretext and the action the sender wants, and returns a verdict with a confidence level.
🔬
3. Deterministic header signals
In parallel, the parser flags sender vs reply-to / return-path mismatch, display-name spoofing, failed SPF/DKIM/DMARC, and every embedded link — facts the model can't talk itself out of.
📎
4. Attachment analysis — static + reputation
Each attachment is statically inspected (true file type, hashes, macro / active-content / double-extension heuristics) and checked against VirusTotal hash reputation and a read-only WildFire verdict lookup — no file is executed.
💥
5. Optional detonation (analyst-initiated)
If you want a live sandbox verdict, the explicit "Detonate" button submits that one attachment to WildFire via XSOAR. Detonation never runs automatically and never happens on this server.
📄
6. Download a report
Export the full analysis as a styled PDF for the ticket or the reporter.
🔒 Runs entirely on the company-owned infrastructure. No email content is persisted; only a who-did-what audit line (analyze / download / detonate) is logged for the SOC trail.
