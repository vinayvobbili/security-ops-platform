How does the Regulatory Acceleration Matrix work?

🎯

The problem it solves

Regulatory changes land in legal/compliance inboxes and stay there. By the time a security team learns that GDPR Art 32 just got tightened or CCPA expanded "sensitive PI", control coverage gaps have already opened. This page is a live crosswalk from regulations to control families, so a single new regulation maps instantly to "which control rows do I need to re-check?"

🧩

1. Crosswalk matrix

Rows are NIST CSF 2.0 control families (placeholder for the company's internal control catalog — that's a v2 question). Columns are regulations. Each cell is colored by coverage state — covered / partial / gap / unknown — and clicking any cell shows the regulation citations and the evidence backing the call.

📡

2. Real evidence where we have it

The LIVE badge means the evidence number came from an actual API at page load (cached 10 min): CrowdStrike fleet size + active detection counts, ServiceNow MIM incident volume. Everything else is DEMO — clearly placeholder, designed to be replaced by Varonis / PAM / DLP / IAM hookups in v2.

🤖

3. AI: regulatory impact diff

The Regulatory Pulse panel surfaces incoming regulatory changes. Clicking Analyze impact sends the change to an LLM with the current control families and asks: "which rows does this update touch, and what action is implied?" Output is a structured JSON answer with rationale + suggested action per affected row. This is the AI wedge. The rest of the product is GRC workflow.

💬

4. Ask the matrix

The chat widget bottom-right has the full crosswalk + live evidence + pulse feed loaded as context. Ask things like "which rows are gap?", "what's HIPAA coverage today?", "which controls back our breach-notification posture?" — answers come from the actual matrix on this page.

🗺️

5. v1 → v2 path

v1 (today): crosswalk shape, two real integrations, AI impact-diff loop, scoping questions surfaced on-page. v2 (after Patrick answers the questions): the company internal control catalog, real regulatory-update feed, live Varonis/PAM/DLP evidence cells, gap → ServiceNow workflow, Walter Auger overlap resolved.
